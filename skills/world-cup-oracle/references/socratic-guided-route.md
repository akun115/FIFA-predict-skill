# 苏格拉底式赛前追问路线（Socratic Pre-Match Guided Route）

供 `world-cup-oracle` skill 引用的可选使用路线。触发条件、设计原则、五段式流程、问题类型表、输出规范和失败退出规则。

---

## 触发条件

当用户表达以下意图时启用本路线，否则走默认预测流程：

- "用苏格拉底式追问带我分析"
- "先别直接给结论"
- "我想自己先判断再看模型"
- "帮我找判断盲点"
- "先让我猜一下"
- 或任何表达"先形成自有判断，再用外部证据/模型校准"意图的说法

## 设计原则

1. **先承诺后校准**：展示 `predict_match` 概率前，让用户给出自己的判断和信心等级，内部记为 `USER_PRIOR`
2. **追问不替代模型**：所有追问只澄清用户判断；`predict_match` 的概率完全来自 Dixon-Coles 数学
3. **证据不绑结论**：联网核验的伤停/阵容等赛前信息，只作为校准参考，不修改概率
4. **分歧是有价值输出**：用户判断与模型概率的分歧不是"谁对谁错"，而是发现盲点的入口
5. **不诱导投注**：全程不得出现下注金额、赔率优势、操作建议

## 五段式路线

### 阶段 1：判断框定（Judgment Framing）

**目标**：帮用户明确要判断什么。

**提问模式**（选 1–2 个）：

- "你想先判断这场比赛的哪个维度？胜平负、比分区间、爆冷风险、晋级概率、战术匹配度，还是模型可能在哪个环节低估了某一方？"
- "是 90 分钟常规时间结果，还是包含加时的晋级概率？"
- "你对这场比赛最感兴趣的问题是什么？"

**输出**：一句清晰的判断对象声明。

### 阶段 2：初始承诺（Initial Commitment / USER_PRIOR）

**目标**：在接触模型任何输出前，让用户给出初始判断。

**提问模式**：

- "在你看来，90 分钟内最可能的结果是什么？用 1–100 给这个判断的信心打分。"
- "你觉得这场比赛最容易被忽略的变量是什么？"
- "如果你必须用一句话解释你的判断，你会说什么？"

**输出格式**（内部）：

```
[USER_PRIOR]
- 判断对象: (胜平负 / 比分区间 / ...)
- 初始判断: (用户原话)
- 信心等级: N/100
- 核心假设: (用户认为支撑其判断的关键原因)
```

### 阶段 3：假设追问（Assumption Probing）

**目标**：用 1–2 个聚焦问题帮用户暴露其判断背后的隐含前提。

**常用问题类型**（按需选，每轮不超过 2 个）：

| 类型 | 示例 |
|------|------|
| 澄清 | "你刚才说的'X 队状态好'，具体指哪场比赛的表现？" |
| 假设 | "你的判断假设 X 队全主力出战——如果 Raphinha 缺阵，你的判断会怎么变？" |
| 反例 | "有什么情况会让你的判断完全反过来？" |
| 影响 | "如果比赛进入加时，你对 120 分钟结果的判断还一样吗？" |

**原则**：
- 不评价用户判断的好坏
- 不问"你确定吗"这类空洞问题
- 追问应指向可核验的事实，而非主观感受

### 阶段 4：证据校准（Evidence Calibration）

**目标**：联网核验赛前事实，帮助用户用证据校准初始判断。

**操作**：
1. 调用 `search_prematch` 生成检索计划
2. 联网搜索伤停、阵容、场地、天气、教练声明、赛程背景
3. 如有 match_id，调用 `get_tournament_state` 获取小组形势

**展示方式**（证据不与判断直接对比，而是并列呈现）：

```
## 证据汇总

| 维度 | 事实 | 来源 |
|------|------|------|
| 伤停 | ... | (链接) |
| 阵容 | ... | (链接) |
| 场地/天气 | ... | (链接) |

### 🧭 与你初始判断的对照

- 你的假设 "[用户核心假设]" 与证据 [一致 / 部分冲突 / 未覆盖]
- [如有冲突]："[具体证据] 和你的预期不同——这对你的判断有影响吗？"
```

**护栏**：
- 不得编造来源
- 证据缺失时说"未获取到该项信息"
- 搜索计划不是证据，必须引用实际网络来源

### 阶段 5：模型对照与收束（Model Comparison & Closure）

**目标**：调用 `predict_match`，对照用户判断、证据和模型概率的异同。

**操作**：
1. 调用 `predict_match`，获取概率、预期进球、Top 5 比分、`model_status`、`model_version`
2. 输出对比表
3. 不做"谁对谁错"的裁判

**输出格式**：

```
## 判断校准摘要

### USER_PRIOR
- 判断: ...
- 信心: N/100 → 校准后: N/100
- 核心假设: ...

### predict_match（模型: national-dc-vX.Y.Z, fitted）
| 维度 | 概率 |
|------|------|
| 主胜 | XX% |
| 平局 | XX% |
| 客胜 | XX% |

### 分歧分析
| 分歧点 | 用户预期 | 模型输出 | 可能解释 |
|--------|---------|---------|---------|
| ... | ... | ... | (伤停/战术/数据盲区等) |

### 收束问题（选 1 个）
- "模型概率和你最初的判断最大的出入在哪里？这让你想到什么？"
- "如果模型有道理，你的判断里哪个假设最值得重新审视？"
- "如果维持你的原判断，模型的哪个限制（如不含伤停输入）最能解释分歧？"
```

**护栏**：
- 概率必须完全来自 `predict_match`，不得手工调整
- 收束问题是邀请反思，不是要求用户认错
- 永远不暗示模型一定正确
- 永远不暗示用户一定正确

## 可用问题类型表

| 类型 | 中文名 | 用途 | 示例 |
|------|--------|------|------|
| Clarify | 澄清 | 消歧义、精确定义 | "你说的'爆冷'，具体指什么？赔率意义上的，还是你主观感觉的？" |
| Probe | 假设追问 | 暴露隐含前提 | "你假设两边都全主力——这个假设的把握有多大？" |
| Evidence | 证据 | 追索可核验事实 | "你的判断有具体的赛前信息支撑吗？还是更多是直觉？" |
| Counter | 反例 | 引入对立视角 | "站在对方球迷的角度，他们会怎么反驳你的判断？" |
| Impact | 影响 | 追溯后果 | "如果你判断错了，最可能的错误来源是什么？" |

## 失败与退出规则

| 场景 | 处理 |
|------|------|
| 用户连续 2 轮拒绝追问 | 跳过阶段 3，直接进入证据校准 |
| 用户要求直接看模型 | 立即退出苏格拉底路线，恢复默认预测流程 |
| 赛前信息无法获取（网络/来源缺失） | 标注数据缺失，不能编造，仍可对照模型 |
| 用户判断为投注话术（提到赔率优势/下注金额） | 声明不提供投注建议，回到分析框架 |
| 5 轮追问后用户判断无变化且拒绝收束 | 自然收束，标注"用户维持初始判断" |
| 用户用中文说"别问了"/"直接给结论" | 立即退出，用默认流程输出 |
| `predict_match` 返回 `provisional` 或 `unseen_team_prior` | 明确展示模型可信度降级，分歧分析中加入此限制 |

## 与默认流程的关系

- 本路线是**可选增强**，不影响默认快速预测流程
- 用户说"预测 X vs Y"时不触发；说"用苏格拉底式追问"时触发
- Agent（Scout/Form/Tactical）仍只做定性解读，概率仍来自 `predict_match`
- `USER_PRIOR` 和校准摘要只服务用户自我认知，不进入模型训练、不回填 `record_prediction`
